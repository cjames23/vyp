# Overrides Export Format

The `vyp-overrides.toml` file format is used to export transitive dependency overrides for library consumers. Libraries ship this file alongside their distribution so that consumers inherit override decisions.

## Overview

Generated by `vyp override export`, this file enables the "middle of the diamond" to propagate override decisions to downstream projects.

---

## Top-Level Fields

| Field | Type | Description |
|-------|------|-------------|
| `overrides-version` | string | Format version (e.g. `"4.0"`). |
| `created-by` | string | Tool and version that generated the file (e.g. `"vyp 0.1.0"`). |
| `package` | string | Package name that generated this file. |
| `package_version` | string | Package version at export time. |
| `overrides` | array of tables | Transitive dependency overrides. |

---

## [[overrides]] Entries

Only **transitive** overrides are exported. Each override entry:

| Field | Type | Description |
|-------|------|-------------|
| `package` | string | Package name to override. |
| `constraint` | string | PEP 440 version specifier. |
| `transitive` | boolean | Always `true` in exported entries. |
| `reason` | string (optional) | Human-readable reason. |
| `origin` | string (optional) | Package that originated this override. |
| `propagation_path` | array of strings | Chain of packages through which this was inherited. |

---

## Example

```toml
overrides-version = "4.0"
created-by = "vyp 0.1.0"
package = "my-library"
package_version = "1.2.0"

[[overrides]]
package = "numpy"
constraint = "==1.24.0"
transitive = true
reason = "resolves GPU/CPU incompatibility in our tree"
origin = "my-library"
propagation_path = ["my-library"]
```

---

## Usage

Consumers of a library that ships `vyp-overrides.toml` can merge these overrides into their own resolution. The format is designed for tooling to read and apply; exact merge semantics are implementation-defined.
